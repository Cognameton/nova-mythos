NOVA-MYTHOS — PHASE 1: FOUNDATION
==================================
Status: IN PROGRESS (started 2026-04-22)

MISSION
-------
Scaffold the nova-mythos project: vendor OpenMythos architecture, establish
project structure, define the integration seam with Nova 2.0, and make the
1B model loadable on hardware.

This phase produces infrastructure only. No trained weights. No Nova 2.0
runtime integration. No training runs.


ARCHITECTURE OVERVIEW
---------------------
Nova 2.0 defines an InferenceBackend protocol (5 methods):
    load(), unload(), metadata(), tokenize(), generate()

LlamaCppBackend is the current implementation (GGUF + llama-cpp-python).
NovaMythosBackend will be the new implementation (PyTorch RDT).

The two projects remain decoupled at import time. Nova 2.0 types are
mirrored locally in backend.py rather than imported cross-project.


OBJECTIVES
----------
1. Initialize git repository on main branch
2. Scaffold directory structure (src, tests, configs, docs, checkpoints)
3. Vendor OpenMythos source (MIT) into src/nova_mythos/model/
   - architecture.py  (from OpenMythos main.py)
   - variants.py
   - tokenizer.py
4. Write config.py (NovaMythosConfig dataclass, load_config())
5. Write backend.py (NovaMythosBackend skeleton — full impl in Phase 3)
6. Write test_hardware.py (Phase 2 gate: VRAM budget, forward pass)
7. Write test_backend.py (Phase 3 gate: protocol compliance)
8. Write pyproject.toml and .gitignore
9. Write nova_mythos.default.yaml

EXIT CRITERIA
-------------
- [ ] Directory structure matches plan

WHAT THIS PHASE DOES NOT DO
